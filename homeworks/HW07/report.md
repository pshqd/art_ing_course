## 2. Protocol

- **Препроцессинг**: ds1/2: **StandardScaler+PCA(10)**; ds2: **drop z_noise**; ds4: **ColumnTransformer(num: mean+Scaler, cat: mode+OHE), drop n25-n30, PCA(20)**
- **Поиск гиперпараметров**:
  - **KMeans**: `k=2..20`, `random_state=42`, `n_init=10/auto`
  - **Agglomerative**: `k=2..20`, `linkage=['ward','complete','average']`
  - **DBSCAN**: `eps=[1.5..3.5]`, `min_samples=[3,5,10]`
  - **Выбор**: максимум **silhouette_score**
- **Метрики**: **silhouette/DB/CH**; DBSCAN — **только non-noise** (label≠-1)
- **Визуализация**: **PCA(2D)** (`random_state=42`) для всех лучших моделей

## 3. Models

**Все датасеты**: **KMeans + AgglomerativeClustering + DBSCAN**

- **KMeans**: `k=2..20`
- **AgglomerativeClustering**: `k=2..20`, `linkage=ward/complete/average`
- **DBSCAN**: `eps=1.5..3.5`, `min_samples=3/5/10`

## 5. Analysis

### 5.1 Сравнение алгоритмов (важные наблюдения)

- **KMeans**: хорошо на простых (ds1 k=2: 0.522), слабее на нелинейных (ds2: 0.467).
- **Agglomerative(ward/average)**: **стабильно лучший** (ds2: 0.507), linkage критично влияет.
- **DBSCAN**: shum~0-1% только при eps=2.0, иначе >90% — датасеты "слишком чистые".
- **Ключ**: **scaling обязательно**, шум (z_noise/n25-n30) портит метрики, PCA снижает размерность.

### 5.2 Устойчивость (обязательно для одного датасета)

- **Проверка**: **5 KMeans(k=6) на ds4** с `random_state=[42,100,200,300,400]`
- **Что получилось**: **ARI stability: 0.95 ± 0.03** (высокая)
- **Вывод**: **Устойчиво** — кластеры воспроизводимы, подтверждает качество k=6.

### 5.3 Интерпретация кластеров

**ds4 (k=6)**: PCA показывает компактные группы. Кластеры различаются по **n1-n24** (высокие/низкие) + **cat_a/b** (A/B vs C-F).  
**Профили**: Кластер 0 — высокие n1-n10+cat_a=A; Кластер 5 — низкие n20+cat_b=Z.  
**Вывод**: Кластеры отражают **типы объектов** по числовым диапазонам+категориям.

## 6. Conclusion

- **Scaling+PCA критично** для всех distance-based методов.
- **Удаление шума** (z_noise, n25-n30) улучшает silhouette на 5-15%.
- **Agglomerative(ward)** универсальнее KMeans при выборе linkage.
- **DBSCAN требует** аккуратного eps (k-distance plot для тюнинга).
- **6 кластеров в ds4** vs 2 в ds1/2 — эффект размерности+категорий.
- **ARI=0.95** подтверждает устойчивость решений.
- **ColumnTransformer** must-have для смешанных данных.
