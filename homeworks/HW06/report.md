## 1. Dataset

  Распределение классов (доля):
  - 0: 0.9508

- Признаки: все признаки числовые (`float64`), столбцы `id` и `target` — `int64`. Пропусков нет. Названия признаков неинформативны, поэтому интерпретация “смысла” признаков затруднена (можно анализировать только важности/вклад).

## 2. Protocol

- Разбиение: train/test  
  Выполнено разбиение `train/test = 0.8/0.2` со `stratify=y` (из-за дисбаланса классов) и `random_state=42`. Тестовая выборка используется только для финальной оценки уже выбранной модели.

- Подбор: CV на train (сколько фолдов, что оптимизировали)  
  Для выбора модели и подбора гиперпараметров использовалась стратифицированная 5-fold кросс-валидация (StratifiedKFold, 5 фолдов) на тренировочной выборке.  
  Для DecisionTree / RandomForest / HistGradientBoosting применялся GridSearchCV с оптимизацией по метрике `f1`. Для baseline-моделей и stacking CV-оценка также считалась на train через `cross_val_score` (scoring=`f1`).

- Метрики: accuracy, F1, ROC-AUC (и почему эти метрики уместны именно здесь)  
  Из-за сильного дисбаланса (примерно 95/5) метрика accuracy может быть высокой даже у модели, плохо находящей положительный класс. Поэтому в качестве основной метрики для сравнения моделей использовалась F1-score (балансирует precision и recall и подходит для дисбалансной бинарной классификации). Дополнительно считались precision/recall и ROC-AUC для оценки качества ранжирования и компромисса между ошибками разных типов.

## 5. Analysis

- Устойчивость: что будет, если поменять `random_state` (хотя бы 5 прогонов для 1-2 моделей) – кратко  
  Устойчивость проверялась на Stacking и HistGradientBoosting при смене `random_state`. Разброс F1 по разным seed оказался небольшим (стандартное отклонение менее 0.01), поэтому вывод: модели достаточно устойчивы к выбору seed на данном датасете.

- Ошибки: confusion matrix для лучшей модели + комментарий  
  Confusion matrix показывает, что модель хорошо распознаёт отрицательный класс, а положительный — заметно хуже, что ожидаемо при сильном дисбалансе классов и малом числе положительных примеров. Для повышения качества по положительному классу можно рассмотреть более агрессивную работу с дисбалансом (например, изменение порога, oversampling/undersampling, подбор class_weight/scale_pos_weight в подходящих моделях).

- Интерпретация: permutation importance (top-10/15) + выводы  
  Для лучшей модели рассчитана permutation importance (top-15). Так как признаки имеют неинтерпретируемые имена, можно лишь перечислить наиболее важные по важности (пример: f54, f25, f53, f58, f38), а график важностей сохранён в артефакты.

## 6. Conclusion

На данных с сильным дисбалансом классов accuracy не является достаточной метрикой качества, поэтому основная метрика сравнения — F1-score. Были обучены и сравнены baseline-модели (DummyClassifier, LogisticRegression) и ансамбли (DecisionTree, RandomForest, HistGradientBoosting, Stacking), причём выбор лучшей модели выполнялся по 5-fold CV на train, а тест использовался только для финальной оценки. Лучший результат показал Stacking с F1 ≈ 0.77 на тестовой выборке; дальнейшее улучшение возможно за счёт более продвинутой работы с дисбалансом и/или улучшения данных.
